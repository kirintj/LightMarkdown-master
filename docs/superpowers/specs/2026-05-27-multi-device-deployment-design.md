# 多端部署设计方案

## 1. 概述

本设计方案旨在为LightMarkdown应用实现多端部署，支持手机、折叠屏和平板设备。基于harmony-next.skills文档中的最佳实践，采用混合布局策略，结合断点系统和栅格系统实现自适应布局。

## 2. 目标设备

- **手机（sm）**：屏幕宽度 [320vp, 600vp)
- **折叠屏（md）**：屏幕宽度 [600vp, 840vp)，支持折叠/展开状态切换
- **平板（lg）**：屏幕宽度 [840vp, ∞)，支持分栏布局

## 3. 架构设计

### 3.1 三层架构

采用文档推荐的三层架构：

1. **Common层**：公共能力（基础库、工具类、通用组件）
   - 已存在：`common/` 模块
   - 新增：断点管理工具类

2. **Feature层**：基础特性（可复用UI模块和业务逻辑）
   - 已存在：`feature/mine`、`feature/markdown`、`feature/star`

3. **Product层**：产品层（针对不同设备形态的HAP）
   - 需要配置：为不同设备配置不同的入口和布局

### 3.2 断点系统

使用以下断点规则：

| 断点名称 | 取值范围（vp） | 设备描述 |
|---------|---------------|---------|
| xs      | [0, 320)      | 最小宽度类型设备 |
| sm      | [320, 600)    | 小宽度类型设备（手机） |
| md      | [600, 840)    | 中等宽度类型设备（折叠屏、小平板） |
| lg      | [840, ∞)      | 大宽度类型设备（平板、PC） |

### 3.3 布局策略

- **手机（sm）**：单列布局，底部Tab导航
- **折叠屏（md）**：可选双列布局，侧边栏可收起
- **平板（lg）**：双栏或三栏布局，侧边栏常驻

## 4. 组件选择

保留原有组件，添加以下适配能力：

1. **断点监听**：在根组件添加窗口监听，计算当前断点
2. **栅格布局**：使用GridRow/GridCol组件实现多列布局
3. **样式适配**：根据断点调整间距、字号等样式

## 5. 实施步骤

### 5.1 第一步：添加断点管理工具类

创建`common/utils/BreakpointManager.ets`，封装断点监听和计算逻辑，提供全局断点状态。

### 5.2 第二步：修改根组件

在`entry/src/main/ets/pages/Index.ets`中引入断点管理，添加窗口监听，将断点状态传递给子组件。

### 5.3 第三步：适配主要页面

- **列表页**：使用GridRow/GridCol实现多列布局
- **详情页**：根据断点调整宽度和边距
- **导航栏**：根据断点切换布局方式

### 5.4 第四步：样式适配

- **间距**：手机12vp，平板24vp
- **字号**：根据设备尺寸微调
- **图片**：根据设备密度加载不同分辨率

### 5.5 第五步：测试验证

使用DevEco模拟器测试不同设备，验证断点切换是否流畅，检查布局在不同尺寸下的表现。

## 6. 预期效果

1. **手机端**：保持原有体验，单列布局，操作便捷
2. **折叠屏**：展开时自动切换到双列布局，提升内容展示效率
3. **平板端**：充分利用大屏空间，展示更多内容，提升阅读体验

## 7. 注意事项

1. **内容连续性**：跨设备切换时任务不中断
2. **性能优化**：避免频繁的布局重计算
3. **兼容性**：确保在不同设备上都能正常运行
4. **测试覆盖**：覆盖所有目标设备类型和断点场景
